# UI Reference (Design Guidelines — NOT Hard Specs)

**Purpose:** Design direction for the Medrelief production build. These HTML files are **reference material only** — Claude should treat them as guidelines, not binding specs.

## Files

| File | Source | Role |
|------|--------|------|
| `medrelief-prototype-2-claude.html` | Generated by Claude | Baseline interpretation of Prototype 2 vision (10 screens) |
| `medrelief-prototype-2-khalid.html` | Khalid's version | Khalid's preferred design direction — use this as the stronger reference |

Both files are mobile-first kiosk designs (390×844 viewport), rendering all key screens of the AI-first patient journey.

## How to use these during production build

When designing or coding a UI screen in production:

1. **Open both HTML files** in a browser. Look at the corresponding screen in each.
2. **Prefer Khalid's version** for style/theme direction when the two disagree.
3. **Extract the patterns, not the markup:**
   - Color palette
   - Spacing + rhythm
   - Card structures
   - Button hierarchy
   - Typography scale
   - Confidence badges / status chips
   - Bottom nav style
4. **Adapt to production needs** — the reference doesn't cover every screen, every state, every edge case. Use judgment.
5. **Do not copy-paste HTML** into production components. Production uses React + Tailwind; these references are static HTML. Rewrite in the target framework.

## What these files are NOT

- Not a spec. If requirements in `docs/prod-plan/*.req.md` contradict the reference, requirements win.
- Not a component library. Production will build its own design system on top of the reference's visual direction.
- Not exhaustive. Many screens (admin panels, settings, error states, empty states, multi-center switching) are not in the reference.
- Not final copy. The Indian-context sample text (Ramesh Kumar, Dr. Mithlesh, ₹1,340) is illustrative — real copy comes from `docs/COPYWRITING.md` + `docs/ANTI AI WRITING STYLE.md` + stakeholder review.

## Design direction summary (distilled)

- **Kiosk-style, touch-first, mobile-first.** 390×844 viewport.
- **Calm, medical, not corporate.** Off-white background `#FAFAF8`. Cards pure white.
- **Primary color: teal `#0891B2`.** NOT cobalt blue. NOT gradients.
- **Generous whitespace.** 24px screen padding.
- **2xl rounded corners** (16–24px radius). Never square.
- **Inter** from Google Fonts. Tight headings.
- **One primary action per screen.**
- **AI confidence badges** on every extracted field (green tick / amber "please confirm").
- **Three equal payment buttons** (CASH / UPI / CARD). Never a dropdown.
- **Manual fallback always visible** on every AI-assisted screen.
- **No jargon.** No "accession", "rate plan", "visit" in UI copy.
- **No emojis in UI.**
- **Lucide-style inline SVG icons.** 18–22px, 2px stroke.
- **WhatsApp green `#25D366`** only for the "Send on WhatsApp" CTA.

See `medrelief-prototype-2-khalid.html` for the canonical visual language.
